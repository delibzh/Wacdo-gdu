    Objectif : Développer le back-office d’une application de borne de commande (Wacdo) en Node.js

    Technos principales : Node.js, Express, MongoDB, JWT, Postman, API REST

    Évaluation finale : Présentation à distance devant un jury pour valider la certification

🍔 Projet Wacdo – Application de commande de type fast-food

C’est une borne de commande avec un back-office pour gérer toute l’activité du restaurant.
🔧 Ce que tu dois construire :

Une API backend complète avec Node.js qui permet de :
1. Gérer les utilisateurs (CRUD)

    Ajouter, modifier, supprimer, lister des utilisateurs

    Sécurité : mots de passe hashés, rôles (admin, équipier accueil, préparateur, etc.)

    Authentification avec JWT

    Middleware pour protéger les routes selon les rôles

2. Gérer les produits et menus (CRUD)

    Ajouter des produits : nom, prix, image, catégorie (boissons, burgers, desserts…)

    Associer des produits à des menus

    Modifier, supprimer, lister

3. Gérer les commandes :

    Création d’une commande via le front (simulé avec Postman ou une maquette Angular)

    Lecture/modification de l’état d’une commande (en préparation, prête, servie)

    Supprimer une commande (admin uniquement)

4. Sécuriser l’API

    Protéger les routes sensibles avec des middlewares (auth et rôle)

    Implémenter les réponses HTTP correctes (codes, messages d’erreur)

    CORS, validation des entrées

5. Base de données MongoDB

    Modélisation des collections :

        User

        Product

        Menu

        Order

    Relations logiques : les commandes contiennent des produits

6. Tests & Documentation

    Tester avec Postman (collections enregistrées)

    Documenter l’API (README ou Swagger facultatif)

    Démo des routes et de leur fonctionnement

7. Version finale

    Tout le code sur GitHub

    Déploiement facultatif sur Render ou Railway
